START THE APP (Choose One)
==========================

1. WITH MOCK BACKEND (Recommended for testing):
   python "frontend (1).py" --mock

2. HEADLESS TEST (Validate system):
   python "frontend (1).py" --mock --no-gui

3. WITH REAL BACKEND:
   Terminal 1: python backend.py
   Terminal 2: python "frontend (1).py"

4. COMPREHENSIVE TEST SUITE:
   python test_all.py

5. CUSTOM BACKEND:
   python "frontend (1).py" --backend-url http://example.com:8080


WHAT YOU GET
============

✅ 60-item dataset (20 books + 20 films + 20 magazines)
✅ Full CRUD (Create, Read, Delete) operations
✅ Filter by category (Book/Film/Magazine)
✅ Search by title or author
✅ Modern light-themed GUI
✅ Mock server (auto-start, in-memory)
✅ Real Flask backend (persistent storage via media.json)
✅ Headless mode for automation/CI


FEATURES
========

FILTER:
  • Select "All", "Book", "Film", or "Magazine"
  • Catalog updates instantly

SEARCH:
  • Type title or author name
  • Press Enter to search
  • Click Refresh to reset

ADD ITEM:
  • Click "+ Add" button
  • Enter: Title, Author, Year, Category
  • Item added to database

DELETE ITEM:
  • Select item in catalog
  • Click "Delete" button
  • Confirm in dialog
  • Item permanently removed

VIEW DETAILS:
  • Click any item in catalog
  • Details show on right panel:
    - ID, Title, Author, Publication Date, Category

TROUBLESHOOTING
===============

Q: Connection Error?
A: Make sure backend is running or use --mock flag

Q: Port already in use?
A: Close other instances or use different port

Q: GUI doesn't open?
A: Remove --no-gui flag or check if already running

Q: Tests fail?
A: Run "python test_all.py" for diagnostics
